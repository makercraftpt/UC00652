Elevador com Abertura de Porta

O modelo base tem 4 estados e a porta "abre instantaneamente". Na realidade existe um motor de porta e um sensor de fim de curso (FC_PORTA) que confirma quando a porta está completamente aberta. Este anexo estende o FSM para incluir esse comportamento.

+1 bit (3 FF-D)
+1 entrada: FC_PORTA
+1 saída: MP (Motor Porta)
+2 self-loops

Motivação

No modelo da Aula 7, ao chegar ao andar o elevador passa diretamente para o estado PISO_X (parado, porta aberta). Na realidade:

  • ▸  A porta tem um motor próprio que precisa de ser ativado
  • ▸  A porta demora algum tempo a abrir (vários ciclos de clock)
  • ▸  Um fim de curso (micro-switch ou sensor ótico) deteta quando a porta está completamente aberta
  • ▸  Só depois do fim de curso é que o sistema passa para o estado de espera
Este é um exemplo perfeito de como uma FSM cresce de forma controlada: acrescentamos 2 estados intermédios (um por andar) e mantemos exatamente a mesma metodologia — diagrama → tabela → equações.
Os 6 estados

Passamos de 2 bits (4 estados) para 3 bits (8 combinações possíveis). Usamos 6; os estados 110 e 111 são inválidos.

Q₂Q₁Q₀ = 000
S0 — PISO_0
P=1 · MS=0 · MD=0 · MP=0
Q₂Q₁Q₀ = 001
S1 — SUBINDO ↑
P=0 · MS=1 · MD=0 · MP=0
Q₂Q₁Q₀ = 010
S2 — PISO_1
P=1 · MS=0 · MD=0 · MP=0
Q₂Q₁Q₀ = 011
S3 — DESCENDO ↓
P=0 · MS=0 · MD=1 · MP=0
Q₂Q₁Q₀ = 100
S4 — PORTA_ABRE_0 ✨
P=0 · MS=0 · MD=0 · MP=1
Q₂Q₁Q₀ = 101
S5 — PORTA_ABRE_1 ✨
P=0 · MS=0 · MD=0 · MP=1
S4 e S5 são os dois estados novos. Em ambos, o Motor da Porta (MP=1) está ativo e o elevador aguarda o fim de curso. A única diferença é o andar onde estão — e por isso têm saídas diferentes para o próximo estado.

6 self-loops (um por estado) + 6 mudanças = 12 linhas. Nenhuma combinação por definir.
Equações do Próximo Estado

Lê-se diretamente das colunas Q₂'Q₁'Q₀' da tabela:

D₂ — bit mais significativo

D₂ = 1 quando o próximo estado pertence a {S4, S5} (Q₂'=1):

S1 → S5: Q₂=0, Q₁=0, Q₀=1, SENSOR=1   → Q₂'·Q₁'·Q₀·SENSOR
S3 → S4: Q₂=0, Q₁=1, Q₀=1, SENSOR=1   → Q₂'·Q₁·Q₀·SENSOR
S4 → S4: Q₂=1, Q₁=0, Q₀=0, FC'         → Q₂·Q₁'·Q₀'·FC_PORTA'
S5 → S5: Q₂=1, Q₁=0, Q₀=1, FC'         → Q₂·Q₁'·Q₀·FC_PORTA'

Simplificado:
D₂ = Q₂'·Q₀·SENSOR  +  Q₂·Q₁'·FC_PORTA'

D₁ — bit intermédio

D₁ = 1 quando o próximo estado tem Q₁'=1 (S2=010, S3=011):

S2 → S2: Q₂=0, Q₁=1, Q₀=0, BTN_0=0   → Q₂'·Q₁·Q₀'·BTN_0'
S2 → S3: Q₂=0, Q₁=1, Q₀=0, BTN_0=1   → Q₂'·Q₁·Q₀'·BTN_0
S3 → S3: Q₂=0, Q₁=1, Q₀=1, SENSOR=0  → Q₂'·Q₁·Q₀·SENSOR'
S5 → S2: Q₂=1, Q₁=0, Q₀=1, FC=1      → Q₂·Q₁'·Q₀·FC_PORTA

Simplificado:
D₁ = Q₂'·Q₁·SENSOR'  +  Q₂·Q₁'·Q₀·FC_PORTA
As linhas S2→S2 e S2→S3 juntas cobrem BTN_0=0 e BTN_0=1 com Q₂'·Q₁·Q₀' — simplificam para Q₂'·Q₁·Q₀' sem depender de BTN_0. Resultado: D₁ = Q₂'·Q₁·(Q₀'+SENSOR') + Q₂·Q₁'·Q₀·FC_PORTA.

D₀ — bit menos significativo

D₀ = 1 quando o próximo estado tem Q₀'=1 (S1=001, S3=011, S5=101):

S0 → S1: Q₂=0, Q₁=0, Q₀=0, BTN_1=1   → Q₂'·Q₁'·Q₀'·BTN_1
S1 → S1: Q₂=0, Q₁=0, Q₀=1, SENSOR=0  → Q₂'·Q₁'·Q₀·SENSOR'
S3 → S3: Q₂=0, Q₁=1, Q₀=1, SENSOR=0  → Q₂'·Q₁·Q₀·SENSOR'
S4 → S4: Q₂=1, Q₁=0, Q₀=0, FC'        → Q₂·Q₁'·Q₀'·FC_PORTA'
S5 → S5: Q₂=1, Q₁=0, Q₀=1, FC'        → Q₂·Q₁'·Q₀·FC_PORTA'

Simplificado:
D₀ = Q₂'·Q₁'·Q₀'·BTN_1 + Q₂'·Q₀·SENSOR' + Q₂·Q₁'·FC_PORTA'

Implementação no LogiSim
  • 1
    3 FF-D + Clock — Memory → D Flip-Flop (×3). Superior = Q₂, médio = Q₁, inferior = Q₀
  • 2
    4 entradas — Wiring → Pin (×4): BTN_1, BTN_0, SENSOR, FC_PORTA
  • 3
    Lógica D₂ — Q₂'·Q₀·SENSOR + Q₂·Q₁'·FC_PORTA'
  • 4
    Lógica D₁ — Q₂'·Q₁·SENSOR' + Q₂·Q₁'·Q₀·FC_PORTA
  • 5
    Lógica D₀ — Q₂'·Q₁'·Q₀'·BTN_1 + Q₂'·Q₀·SENSOR' + Q₂·Q₁'·FC_PORTA'
  • 6
    Saídas Moore → LEDs — P=NOT(Q₂)·NOT(Q₀) · MS=NOT(Q₂)·NOT(Q₁)·Q₀ · MD=NOT(Q₂)·Q₁·Q₀ · MP=Q₂·NOT(Q₁)

    Guardar como elevador_porta.circ e testar a sequência completa — verificar que MP acende durante a abertura de porta
